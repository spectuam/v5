# v5 日更自动化方案 + 手动应急预案

## 自动链路

```
每天 15:35 timer 触发:

Step 1 — importdata (Hikyuu)
  从 pytdx 直连通达信服务器, 拉最近 7 天日线 (不复权)
  → 写入 HDF5 (~/stock/sh_day.h5, sz_day.h5)
  预计 30~60s
  失败处理: 重试 1 次, 仍失败则发告警, 跳过今天

Step 2 — 桥脚本 hdf5_to_sqlite.py
  2a. 从 HDF5 读最近 7 天的数据 (不复权 OHLCV)
  2b. 三重过滤: class='stock' + name 不含 ST + code 不含 bj
  2c. × 复权因子 → 后复权 open/high/low/close
      - 成交量 / 复权因子 → 后复权 volume
  2d. INSERT OR IGNORE INTO daily_kline
      主键 (code, date) 保证幂等 — 重复跑不会重复插
  预计 1~2min

Step 3 — 检查 (可选)
  SELECT COUNT(*), MAX(date) FROM daily_kline
  与预期对比: 如果最后日期 < 昨天 → 告警

Step 4 — 复权因子更新 (每周日 02:00)
  对每个股票:
    复权因子 = daily_kline 后复权 close / HDF5 不复权 close
             (取最近一个两者都有数据的交易日)
  存入 adjustment_factor 表 (code, factor, updated_at)
  如已存在则 UPDATE, 否则 INSERT
```

## 数据防乱设计

| 原则 | 实现 |
|------|------|
| 幂等 | `INSERT OR IGNORE`, 主键 (code, date) |
| 无部分状态 | 每个交易日的数据一次性 INSERT, 不在中间状态 |
| 可回放 | 可以指定任意日期范围重跑, 已存在的跳过 |
| 可追溯 | 每条记录有 source 字段 (HDF5/TDX/Baostock) |

## 手动应急预案

### 场景 1: importdata 挂了, HDF5 没更新
  症状: HDF5 最新日期 < 昨天
  处理: 
    1. 等 TDX 服务器恢复 (通常几小时)
    2. 手动跑: `~/v5/.venv/bin/importdata`
    3. 然后手动跑桥脚本: `python3 hdf5_to_sqlite.py --days 7`
  影响: 0 — 纯粹延迟, 数据不会错

### 场景 2: 桥脚本挂了, HDF5 有数据但没进 SQLite
  症状: HDF5 有昨天, SQLite 没有
  处理:
    1. 直接重跑桥脚本: `python3 hdf5_to_sqlite.py --days 7`
    2. INSERT OR IGNORE 保证不会重复
  影响: 0

### 场景 3: 数据库被污染 (某天价格不对)
  症状: 发现某天数据异常
  处理:
    1. 找出异常日期: `SELECT date, COUNT(*) FROM daily_kline WHERE date='xxx'`
    2. 删掉那天: `DELETE FROM daily_kline WHERE date='xxx'`
    3. 重跑桥脚本补那天: `python3 hdf5_to_sqlite.py --dates xxx`
  影响: 只丢了那天的数据, 补回来后恢复

### 场景 4: WSL2 崩溃 (中途被杀)
  症状: 什么状态都有可能
  处理:
    1. 先查最后完整日期: `SELECT MAX(date) FROM daily_kline`
    2. 再查 HDF5 最后日期: hikyuu API
    3. 从最后完整日期的后一天开始补: `python3 hdf5_to_sqlite.py --since <日期>`
    4. INSERT OR IGNORE 跳过已存在的
  影响: 0

### 场景 5: 连续多天挂了
  症状: 缺了好几天
  处理:
    1. `importdata` 一次 (它自动拉最近 7 天)
    2. `python3 hdf5_to_sqlite.py --days 7`
    3. 7 天全覆盖, 缺的自然补上
  影响: 0

## 关键规则 (给 CC 自己记)

- **永远不要 UPDATE daily_kline。** 错了就 DELETE + 重 INSERT。
- **补数据时不要删全库。** 只删需要补的日期范围。
- **跑桥脚本前确认 HDF5 有数据。** 否则会清空那几天 (INSERT 0 rows)。
- **日更失败继续跑第二天。** 缺几天不是灾难, 补回来就行。
- **复权因子只在周日更新。** 新的除权事件要等下一个周日才会反映到后复权价格中。
  这会导致除权日后几天的价格有轻微偏差, 但下一个周日自动修正。
