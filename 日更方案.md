# v5 日更数据通道方案

> 更新: 2026-07-19

## 架构

```
每天 15:05 自动              老板不定期手动 (1-2周一次)
─────────────                ─────────────────────────
Sina API (主)                Windows: tdx2db.exe cron
34秒 ~5535只                 ↓ DuckDB 更新
     │                       WSL: tdx_vipdoc_sync.py
     │                       ① update_stock_info()    新股上市
     │                       ② update_adjustment_factors()  复权因子
     │                       ③ sync_vipdoc()  日K补全
     │                            │
     └────────────┬───────────────┘
                  ▼
           tdx_stock_data.db
           daily_kline (后复权)
           stock_info (class='stock')
           adjustment_factor (后复权因子)
                  │
                  ▼
           build_daily_panel() SQL过滤
           → 因子计算 → daily_pick_v5
```

## 数据规则

| 项目 | 规则 |
|------|------|
| 股票范围 | stock_info 中 `class='stock'` (约 5535 只, 含 ST/BJ) |
| 价格类型 | 后复权 (不复权源 × hfq_factor) |
| 日期格式 | `YYYY-MM-DD 00:00:00` |
| 价格精度 | 保留 2 位小数 |
| 主键 | (code, date) |
| 写入方式 | INSERT OR REPLACE (幂等) |

## 主通道: Sina API

- **脚本**: `~/v5/sina_daily_sync.py`
- **定时器**: `sina-daily-sync.timer`, 每天 15:05
- **数据源**: `hq.sinajs.cn` HTTP API, 100 只/批次
- **处理**: 不复权 OHLCV → × hfq_factor → 后复权
- **耗时**: ~35 秒
- **职责**: 当日日K, 轻量稳定, 保证 90 天窗口可用
- **不管**: stock_info、复权因子、历史回补

## 后备通道: TDX vipdoc

- **脚本**: `~/v5/tdx_vipdoc_sync.py`
- **触发**: 老板手动, 1-2 周一次
- **前置**: Windows 端运行 `tdx2db.exe cron --dburi duckdb://./tdx.db` 更新 DuckDB
- **数据源**: `/mnt/c/new_tdx64/vipdoc/{sh,sz,bj}/lday/*.day` 二进制
- **处理**: 读 .day 二进制 → × hfq_factor → 后复权
- **额外职责**:
  - 扫 vipdoc 新股 → INSERT INTO stock_info
  - 从 DuckDB raw_adjust_factor → 更新 SQLite adjustment_factor
- **用法**:
  ```bash
  ~/v5/.venv/bin/python3 ~/v5/tdx_vipdoc_sync.py --dates 2026-07-16,2026-07-17
  ~/v5/.venv/bin/python3 ~/v5/tdx_vipdoc_sync.py --since 2026-07-16
  ```

## 面板层过滤

`build_daily_panel()` 统一过滤 (不影响数据库存储):
```sql
JOIN stock_info s ON d.code = s.symbol
WHERE s.class = 'stock'
  AND s.name NOT LIKE '%ST%'
  AND d.code NOT LIKE 'bj%'
```

## 应急预案

| 场景 | 处理 |
|------|------|
| Sina 挂了/缺数据 | 有后备, 不影响。事后老板跑一次 vipdoc 补 |
| 多个交易日缺数据 | `tdx_vipdoc_sync.py --since <日期>`, INSERT OR REPLACE 幂等 |
| 某天数据异常 | DELETE FROM daily_kline WHERE date='...' → 重跑 vipdoc |
| 新股上市未覆盖 | 等下次 vipdoc sync, stock_info 自动更新 |
| 复权因子过期 | 老板 Windows 跑 `tdx2db cron` → 跑 vipdoc sync |
